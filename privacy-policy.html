📋 一、概述

SafeNet（以下簡稱「本應用程式」）是一款無償互助求救平台，旨在幫助使用者在需要時發送求救訊號，並讓附近的善心人士能夠前往協助。

我們深知您的隱私至關重要。本隱私權政策說明我們如何收集、使用、儲存及保護您的個人資料，並告知您依據相關法律所享有的權利。

適用範圍：本政策適用於 SafeNet 在 iOS（Apple App Store）及 Android（Google Play Store）平台上發布的所有版本。

📍 三、位置資料（GPS）使用說明

⚠️ 重要說明：本應用程式會在背景持續取得您的 GPS 位置。

為什麼需要位置資料？

  • 發布求救訊號：將您的位置標註在地圖上，讓附近的人能找到您。
  • 顯示附近求救：在地圖上顯示您周圍的求救標記，讓您可以前往幫助他人。
  • 緊急求救定位：在緊急情況下即時提供您的位置，提高救援效率。

前景定位與背景定位

定位模式 說明 使用時機
前景定位 App 開啟使用中時取得位置 瀏覽地圖、發布求救
背景定位 App 在背景或螢幕關閉時持續取得位置 持續更新您的位置以便他人在緊急情況下找到您

您的控制權

  • 您可以隨時在裝置設定中將定位權限改為「使用期間」或「永不」。
  • 拒絕背景定位不影響基本功能，但可能降低緊急求救時的定位準確度。
  • iOS：設定 → 隱私權與安全性 → 定位服務 → SafeNet
  • Android：設定 → 應用程式 → SafeNet → 權限 → 位置

🔔 四、推播通知

本應用程式使用 Firebase Cloud Messaging（FCM）發送推播通知。

通知用途

  • 緊急求救通知：當您附近有人發送緊急求救時，立即通知您。
  • 一般求救通知：當您附近有人發送一般求救時通知您。
  • 狀態更新：您發布的求救請求被回應或狀態變更時通知您。

技術說明

  • 我們會在您的裝置上產生一組 FCM Token，用於定向發送通知。
  • 此 Token 與您的帳號關聯，儲存在我們的伺服器中。
  • 登出時，Token 將從伺服器移除。
如何關閉通知：
  • iOS：設定 → 通知 → SafeNet → 關閉「允許通知」
  • Android：設定 → 應用程式 → SafeNet → 通知 → 關閉

🎯 六、資料使用目的

  • 提供核心服務：發布與顯示求救訊號、地圖定位、推播通知。
  • 帳號管理：建立與維護您的帳號、記錄偏好設定。
  • 安全與防濫用：記錄 IP 位址與時間，防止惡意濫用求救功能。
  • 服務改善：分析匿名化的使用趨勢以改善應用程式體驗。
🚫 我們絕不會：
  • 將您的個人資料出售給第三方
  • 將位置資料用於廣告投放
  • 未經同意向第三方分享您的可識別資料
  • 對您的資料進行自動化決策或行為分析

🔒 七、資料儲存與安全

項目 說明
儲存位置 Supabase 雲端資料庫（基於 PostgreSQL），伺服器位於受保護的資料中心
傳輸加密 所有資料透過 HTTPS/TLS 加密傳輸
存取控制 採用 Row Level Security（RLS），使用者僅能存取自己的資料
求救紀錄保留 活躍的求救訊號於 24 小時後自動過期
FCM Token 登出時自動從伺服器移除

🗑️ 九、帳號刪除與資料清除

⚠️ 帳號刪除為不可逆操作。

一旦刪除，以下所有資料將被永久清除且無法復原。

如何刪除帳號

在應用程式內操作：設定頁面 → 刪除帳號 → 確認刪除

刪除時清除的資料

  • ✅ 個人檔案（姓名、電子郵件、大頭貼）
  • ✅ 所有求救發布紀錄（包含備註、位置、IP 紀錄）
  • ✅ FCM 推播 Token
  • ✅ 偏好設定與語言選擇
  • ✅ 系統內部帳號編號
  • ✅ 身分驗證資料

處理時間

帳號刪除請求將立即執行。所有資料將在操作完成後從資料庫中永久移除。

👶 十、兒童隱私保護

本應用程式不針對 13 歲以下兒童提供服務。我們不會故意收集 13 歲以下兒童的個人資料。如果您是家長或監護人，並發現您的孩子在未經同意的情況下向我們提供了個人資料，請與我們聯繫，我們將立即刪除相關資料。

🍪 十一、Cookie 與追蹤技術

本應用程式不使用 Cookie、廣告追蹤碼或第三方分析工具。我們僅使用以下技術：

  • Supabase Auth Token：用於維持登入狀態的安全 Token。
  • SharedPreferences / UserDefaults：在裝置本地儲存語言偏好設定。
  • Firebase Cloud Messaging：僅用於推播通知功能。

📝 十二、隱私權政策變更
